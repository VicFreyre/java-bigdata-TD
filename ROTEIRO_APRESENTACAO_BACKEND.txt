ROTEIRO DE APRESENTAÇÃO DO SISTEMA DE MATRÍCULAS

1. INTRODUÇÃO AO APP
O Sistema de Matrículas é uma aplicação web desenvolvida em Java com Spring Boot, voltada para o gerenciamento completo de alunos, cursos e matrículas. Possui interface moderna, relatórios e integrações com banco de dados PostgreSQL.

2. BACKEND: CONCEITOS FUNDAMENTAIS
- Classe: Estrutura que define atributos e comportamentos de um tipo de objeto. Exemplo: `Aluno`, `Curso`, `Matricula`.
- Objeto: Instância de uma classe, representando um elemento real do sistema. Exemplo: Um aluno cadastrado.
- Método: Função definida dentro de uma classe, responsável por executar ações. Exemplo: `cancelar()`, `trancar()` em `Matricula`.

Exemplo de classe e método:
```java
public class Aluno {
    private Long id;
    private String nome;
    // ...
    public void atualizarEmail(String novoEmail) {
        this.email = novoEmail;
    }
}
```

3. FLUXO DE FUNCIONAMENTO
- Controladores: Recebem requisições HTTP, validam dados e encaminham para os serviços. Exemplo: `AlunoControlador`, `MatriculaControlador`.
- Serviços: Implementam regras de negócio, como validação, trancamento ou reativação de matrícula. Exemplo: `MatriculaServicoImpl`.
- DTOs (Data Transfer Objects): Transportam dados entre camadas, evitando exposição direta das entidades. Exemplo: `AlunoDTO`, `MatriculaDTO`.

Fluxo:
Usuário → Controller → Service → Repository → Banco de Dados

Exemplo de controlador:
```java
@PostMapping("/atualizar")
public String atualizar(@RequestParam Long id, @RequestParam String acao, RedirectAttributes redirectAttributes) {
    if ("trancar".equals(acao)) {
        matriculaServico.trancar(id);
    }
    // ...
    return "redirect:/matriculas";
}
```

Exemplo de serviço:
```java
public void trancar(Long id) {
    Matricula matricula = matriculaRepositorio.findById(id)
        .orElseThrow(() -> new RecursoNaoEncontradoException("Matrícula não encontrada"));
    matricula.trancar();
    matriculaRepositorio.save(matricula);
}
```

Exemplo de DTO:
```java
public class AlunoDTO {
    private Long id;
    private String nome;
    private String email;
    // ...
}
```

4. MÓDULOS DO SRC
- controlador: Controladores REST, recebem e tratam requisições.
- entidade: Entidades JPA, representam as tabelas do banco de dados.
- repositorio: Repositórios JPA, realizam operações de acesso ao banco.
- servico: Interfaces de serviços, definem regras de negócio.
- servico/impl: Implementações dos serviços.
- dto: Objetos de transferência de dados.
- excecao: Exceções customizadas para tratamento de erros.
- relatorio: Serviços para geração de relatórios.

5. EXEMPLOS ILUSTRATIVOS
Classe entidade:
```java
@Entity
public class Matricula {
    @Id
    private Long id;
    private String status;
    public void trancar() { this.status = "TRANCADA"; }
}
```

Controlador:
```java
@GetMapping("/editar/{id}")
public String editar(@PathVariable Long id, Model model) {
    MatriculaDTO dto = matriculaServico.buscarPorId(id).get();
    model.addAttribute("matricula", dto);
    return "matriculas-editar";
}
```

DTO:
```java
public class MatriculaDTO {
    private Long id;
    private String status;

}
```

6. CONCLUSÃO
O backend do sistema é robusto, modular e segue boas práticas de POO e arquitetura em camadas, facilitando manutenção, testes e evolução do sistema.
